# ツール契約

## 前提
- `agents/rules.md` が最上位のSSOT
- ツールは `探索 -> 局所編集 -> 検証` の順で使う

## 読み取りの原則
- ファイル検索は `rg --files`、テキスト検索は `rg` を優先する
- 読み取り系コマンドは並列化を優先する
- まず非破壊の探索で事実を集める

## 書き込み可能
- `docs/`
- `scripts/`
- 将来 `src/` などの実装領域（ただし承認済みPRDがある場合）

## 編集の原則
- 編集は局所的に行う
- 既存のユーザー差分は戻さない
- 破壊的コマンドより、明示的で可逆な編集を優先する

## 禁止
- Secrets/PII の生成、出力、埋め込み
- 外部送信設定の追加（明示許可なし）
- `git reset --hard` などの破壊的Git操作（明示承認なし）

## Evidence 要件
- 変更には根拠と検証手順を添付する
- 根拠は差分、ログ、テスト、計測、仕様参照のいずれかで示す

## Skill 化の目安
- 同じ探索または検証手順を 2 回以上繰り返した
- 入力と出力が安定している
- 成否を評価しやすい
